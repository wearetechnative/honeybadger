## Context

The `RUNME.sh audit` command currently converts Lynis reports to JSON using a Docker container running the legacy Perl-based `lynis-report-converter.pl`. This requires Docker to be installed and results in a large image build on first run. The upstream `wearetechnative/lynis-report-converter` repo now ships a simple Python script (`lynis_report_converter.py`) that does the same JSON conversion with no extra dependencies beyond Python 3.

The Lynis report is always at `/var/log/lynis-report.dat` (generated by root). The converter reads this `.dat` file and outputs JSON.

## Goals / Non-Goals

**Goals:**
- Make the Python script the default converter in `RUNME.sh audit`
- Keep Docker as a configurable fallback via `USE_DOCKER_CONVERTER=true` in `honeybadger.conf`
- Remove `docker` from required dependency checks when using Python mode
- Minimize changes to the overall audit flow

**Non-Goals:**
- Rewriting the Docker-based converter or Dockerfile
- Supporting output formats other than JSON (the Python script only does JSON)
- Auto-installing Python or pip if missing

## Decisions

### 1. Bundle the Python script in the repo

**Decision**: Clone/download `lynis_report_converter.py` into `lib/lynis_report_converter.py` and commit it to the honeybadger repo.

**Rationale**: Fetching at runtime adds network dependency and failure modes during audit. The script is small and owned by the same org. Bundling keeps the audit fully offline-capable.

**Alternative considered**: Git submodule or runtime download — rejected for simplicity and reliability.

### 2. Config key `USE_DOCKER_CONVERTER`

**Decision**: Add `USE_DOCKER_CONVERTER=false` to `.honeybadger.conf.example`. When `true`, the existing Docker flow runs. When `false` (default), the Python script runs.

**Rationale**: Simple boolean, clear name, backward-compatible default (Python is the new default per the user's request). Users who need the legacy Perl converter's extra formats can opt in.

### 3. Dependency check conditional on converter mode

**Decision**: Only `checkdeps "docker"` when `USE_DOCKER_CONVERTER=true`. Always `checkdeps "python3"` (or fall back to `python`) when using Python mode.

**Rationale**: Avoids false dependency failures on systems without Docker.

### 4. Python script invocation

**Decision**: Run `python3 "$thisdir/lib/lynis_report_converter.py" --input-path /var/log/lynis-report.dat --output-path "$output/lynis-report.json"`.

**Rationale**: Matches the documented CLI of the new converter. Writes directly to the output file instead of piping stdout.

## Risks / Trade-offs

- **[Python not installed]** → Audit fails with clear error from `checkdeps`. Most Linux systems have Python 3.
- **[Script diverges from upstream]** → Bundled copy may get stale. Mitigation: document upstream URL, periodically update.
- **[Docker users surprised by default change]** → Mitigation: document in README/changelog; config fallback is easy.
